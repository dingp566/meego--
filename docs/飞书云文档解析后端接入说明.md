# 飞书云文档解析后端接入说明

## 背景

交付物插件前端已经支持“飞书云文档链接”作为版本来源，并预留了后端解析接口：

```http
POST /api/proxy/lark-documents/parse
```

当前纯前端只能保存 URL，无法稳定读取云文档标题和正文。原因是：

- 用户输入的纯 URL 本身不包含文档标题。
- 浏览器前端不能跨域读取飞书云文档页面内容。
- 飞书项目 dashboard 插件 JSSDK 当前没有暴露原生 URL 字段控件的解析标题能力。
- `lpm perm list` 显示当前飞书项目插件权限覆盖工作项、附件、评论、配置等 Meegle OpenAPI，没有云文档标题/正文读取能力。

因此需要一个自建后端或企业内部网关，使用飞书开放平台/云文档侧能力解析 URL，再返回给插件前端。

## 前端接口契约

### 请求

```http
POST /api/proxy/lark-documents/parse
Content-Type: application/json
```

```json
{
  "url": "https://bytedance.larkoffice.com/docx/xxxx"
}
```

### 响应

```json
{
  "title": "交付物管理插件 - PRD v0.1",
  "content": "正文第一段\n正文第二段\n正文第三段"
}
```

字段说明：

| 字段 | 必填 | 说明 |
|---|---:|---|
| `title` | 否 | 云文档标题。返回后会作为版本卡片名称。 |
| `content` | 否 | 云文档纯文本正文。返回后会作为版本 Diff 的对比内容。 |

如果后端暂时只能返回标题，也可以只返回：

```json
{
  "title": "交付物管理插件 - PRD v0.1",
  "content": ""
}
```

## 前端行为

前端在用户点击“确认链接”时会按以下优先级生成版本名称：

1. 后端接口返回的 `title`
2. 从剪贴板富文本链接里识别出的标题
3. 兜底名称：`飞书云文档链接`

版本 Diff 正文来源：

1. 后端接口返回的 `content`
2. 无 `content` 时保存为空文本，Diff 不展示正文变化

## 后端实现建议

### 解析 URL

后端需要从 URL 中识别文档类型和 token，例如：

| URL 类型 | 示例路径 | 说明 |
|---|---|---|
|新版文档| `/docx/<token>` | 需要读取文档标题和正文块内容 |
|Wiki| `/wiki/<token>` | 可能需要先解析到实际文档 token |
|旧版文档| `/doc/<token>` | 视租户启用情况支持 |

### 权限

后端不应使用前端 bundle 内的任何密钥。建议由后端环境变量维护飞书开放平台应用凭据，例如：

```bash
LARK_APP_ID=...
LARK_APP_SECRET=...
```

后端需要具备读取云文档元信息和正文内容的权限。具体 scope 以飞书开放平台文档和企业审批为准。

### 返回正文格式

为了适配当前前端 Diff，建议把云文档内容转为纯文本段落：

```text
一级标题
正文段落 A
正文段落 B
表格：字段1 | 字段2 | 字段3
```

处理建议：

- 标题、段落、列表项各占一行。
- 表格可转为多行文本，保留关键单元格内容。
- 图片、附件、卡片等非文本块可转为占位说明，例如 `[图片] 架构图`。

## 错误处理

| 场景 | HTTP 状态 | 建议返回 |
|---|---:|---|
| URL 非法 | 400 | `{ "message": "invalid document url" }` |
| 无权限访问 | 403 | `{ "message": "permission denied" }` |
| 文档不存在 | 404 | `{ "message": "document not found" }` |
| 飞书 API 异常 | 502 | `{ "message": "failed to parse document" }` |

前端当前会在接口失败时静默保存链接版本，不阻断用户上传。

## 联调步骤

1. 后端启动并暴露：

```http
POST /api/proxy/lark-documents/parse
```

2. 在本地调试环境中配置代理，使插件 iframe 能访问该接口。

3. 打开飞书项目插件调试：

```bash
lpm --cwd /Users/bytedance/Codex-二开插件/未命名文件夹/plugin-03cbb8 start --auto
```

4. 在交付物插件中切换到“飞书云文档链接”，输入 URL，点击“确认链接”。

5. 预期结果：

- 版本卡片名称显示后端返回的 `title`
- 点击版本可看到 `content` 转成的正文阅览
- 两个云文档版本之间可以进行 Diff
