# 交付物版本管理插件操作说明

## 1. 插件简介

交付物版本管理插件用于在飞书项目工作项详情页中管理 DOCX 交付物。插件支持上传交付物、保存版本、设置基线版本、阅览与下载附件，并对两个版本进行正文差异对比。

当前版本主要支持 DOCX 文件，适用于项目交付文档、可行性分析报告、设计文档、方案文档等需要沉淀版本记录的场景。

## 2. 入口位置

1. 打开飞书项目。
2. 进入目标空间和项目。
3. 打开一个工作项详情页。
4. 在详情页顶部 Tab 中点击「交付物」。
5. 进入交付物版本管理页面。

如果未看到「交付物」Tab，请确认插件已安装并在当前空间启用。

## 3. 页面结构

页面主要分为四个区域：

| 区域 | 说明 |
| --- | --- |
| 顶部标题区 | 显示当前工作项名称和当前基线版本 |
| 上传区 | 上传新的 DOCX 文件，确认后生成新版本 |
| 已上传附件 | 展示基线版本和历史版本，支持阅览、下载、打基线和删除 |
| 版本对比 | 选择两个版本进行正文 Diff，并通过变更导航快速查看差异 |

## 4. 上传新版本

1. 在「上传新版本」区域点击上传框，或将 DOCX 文件拖拽到上传框中。
2. 插件会解析 DOCX 正文内容。
3. 上传后页面会出现「附件编辑中，确认后完成更新」提示。
4. 点击「确定」后，插件会保存文件并生成新版本。
5. 点击「取消」则放弃本次上传。

版本号会自动递增，例如：

- 第一次上传生成 V1
- 第二次上传生成 V2
- 第三次上传生成 V3

## 5. 已上传附件管理

上传成功后，文件会出现在「已上传附件」区域。

每个附件卡片包含：

- Word 文件图标
- 文件名称
- 版本号
- 基线状态
- 当前版本状态
- 文件大小
- 上传时间
- 操作按钮

### 5.1 基线版本

基线版本会固定展示在附件列表最上方。

基线版本通常代表当前正式确认的交付物版本，后续版本可以与基线版本进行对比。

### 5.2 历史版本

除基线版本外，其他版本会按两列排列展示。超过一行时自动换行。

## 6. 阅览附件

点击附件卡片中的文件信息区域，可以打开 DOCX 阅览窗口。

阅览窗口中可以查看插件解析出的正文内容，并可点击「下载附件」获取原始 DOCX 文件。

如果历史版本是在早期调试阶段上传，可能缺少原始文件数据。此时建议重新上传该版本文件。

## 7. 下载附件

点击附件卡片右侧的下载图标，可以下载该版本的原始 DOCX 文件。

下载能力受平台权限和插件运行环境影响。如果无法下载，请确认当前用户具有对应工作项和附件的访问权限。

## 8. 设置基线版本

点击附件卡片右侧的星标图标，可以将该版本设置为基线版本。

设置后：

1. 顶部「基线版本」会更新为对应版本号。
2. 该版本会固定展示在附件列表最上方。
3. 版本对比区域会默认以基线版本作为对比对象。

已经是基线的版本不能重复设置为基线。

## 9. 删除版本

点击附件卡片右侧的垃圾桶图标，可以删除该版本。

删除后：

- 该版本会从附件列表中移除。
- 如果删除的是基线版本，系统会自动选择剩余版本中的最新版本作为基线。
- 被删除版本不再参与版本对比。

删除操作请谨慎使用。

## 10. 版本对比

版本对比区域用于比较两个 DOCX 版本的正文差异。

### 10.1 选择对比版本

1. 在左侧下拉框选择旧版本。
2. 在右侧下拉框选择新版本。
3. 默认情况下，插件会以基线版本对比当前版本。

### 10.2 汇总视图

「汇总」模式会聚合展示所有发生变化的段落，包括：

- 新增内容
- 删除内容
- 修改内容

适合快速了解本次版本变化。

### 10.3 对比视图

「对比」模式会以左右两列展示旧版本和新版本内容。

适合逐段检查两个版本之间的差异。

### 10.4 变更导航

右侧「变更导航」会统计本次对比的变更数量，并支持点击跳转到对应变更位置。

统计项包括：

- 新增
- 修改
- 删除

## 11. 推荐使用流程

建议团队按以下流程使用：

1. 首次上传正式交付物，生成 V1。
2. 将 V1 设置为基线版本。
3. 后续每次更新文档时上传新 DOCX，生成 V2、V3 等版本。
4. 在版本对比中查看新版本相对基线版本的变化。
5. 评审通过后，将新版本设置为新的基线版本。

## 12. 常见问题

### 12.1 为什么只能上传 DOCX？

当前版本聚焦 DOCX 交付物，系统会解析 DOCX 正文用于版本 Diff。其他文件类型暂未纳入本版本范围。

### 12.2 上传后为什么要点击确定？

上传文件后，插件会先进入编辑确认状态。只有点击「确定」后，才会正式保存版本。这样可以避免误上传或上传错误文件。

### 12.3 为什么有些旧版本无法下载？

如果旧版本是在早期调试阶段上传，可能只保存了文本内容，没有保存原始 DOCX 文件。重新上传后即可正常下载。

### 12.4 Diff 对比为什么和 Word 修订模式不完全一样？

插件当前基于 DOCX 正文段落做差异比较，重点用于快速识别交付物内容变化。它不是 Word 原生修订模式，也不会解析所有复杂格式。

### 12.5 为什么看不到交付物 Tab？

请检查：

- 插件是否已安装
- 插件是否在当前空间启用
- 当前工作项类型是否允许展示该插件
- 是否还处于插件调试模式

## 13. 注意事项

- 当前版本以浏览器本地能力保存演示数据，正式生产持久化建议接入后端或平台字段存储。
- 删除版本前请确认不再需要该版本。
- 设置基线后，版本对比默认会围绕基线展开。
- DOCX 内容过于复杂时，解析结果可能与 Word 原始排版存在差异。
